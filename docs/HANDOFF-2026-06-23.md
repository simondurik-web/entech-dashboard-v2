## How to work here (so reviews + deploys stay consistent)
- **Every code change → fleet review before pushing** (project rule, even small fixes): Codex
  (`codex exec --skip-git-repo-check "<prompt>" < /dev/null`), Grok (`grok --file <diff> "<prompt>"`,
  often hangs — proceed on the others), Gemini (`handoff gemini` — frequently times out), + an Opus
  sub-agent. Start verdicts with `VERDICT: APPROVE|BLOCK`.
- **Promote staging → main** (after Simon's OK): `git push origin "$(git rev-parse origin/staging)":refs/heads/main`
  (the pre-push hook blocks pushes only when the current branch is `staging`; we're on `feat/label-redesign`).
- **Build check:** `npm run build` "Compiled successfully" is the signal; it then fails at page-data
  collection with `supabaseUrl is required` (env not loaded locally) — that's expected, not a real error.
  Typecheck: `npx tsc --noEmit` (ignore pre-existing push-notifications/register-sw/sales-math.test errors).
- **GitNexus impact** before editing shared symbols. The ERP↔dashboard wiring is isolated in
  `lib/erpnext/*` + `app/api/erpnext/*`; the shared glue (nav entry, `/inventory-ops` permission, auth,
  `inventory_ops_log`/`print_jobs`) is documented in the repo `CLAUDE.md` "Connections to other systems".
- **Bench scripts** run as: `cd ~/frappe-bench/sites && ~/frappe-bench/env/bin/python /tmp/<script>.py`
  (script does `frappe.init(site="erp.local", sites_path=".")` + `frappe.connect()`).

## Reference: ERPNext gotchas hit this session
- Convert to non-batch only at 0 stock / 0 Stock Ledger Entries (ERPNext blocks the toggle otherwise).
- Query Sales Orders by item with a **parent query + child-table filter**
  (`["Sales Order Item","item_code","=",code]` on `Sales Order`) — querying the child doctype directly
  trips Frappe's parent-permission check (403).
- All user free-text into Stock Entry `remarks` is bracket-stripped (`safeRemark`) so it can't smuggle an
  `[op:<key>]` token that would poison `reconcileStockEntry`.
